We're excited to share the latest features that have been rolled out in January and so far in February! This update focuses on improving field worker experience with better navigation capabilities, enhanced issue reporting, and improvements related to data uploads.

### 📍 Location Navigation

Field workers can now easily save and navigate to beneficiary locations. When at a beneficiary's residence, field workers can click the add location icon to save the beneficiary's location. Once saved, they can tap the location icon to navigate to it anytime.

With offline maps downloaded, field workers can navigate to saved locations even without an internet connection—ensuring uninterrupted fieldwork in remote areas. Refer [here](https://avni.readme.io/docs/record-location-of-a-subjectindividual) for more details.

### 🛠️ Ability to Report Issues

We’ve made it easier for users to share information about the issues they face. When a user encounters an issue, they can now click "Upload issue info" on the error alert that appears. This automatically uploads diagnostic logs to our server, helping us resolve issues faster.

<div style="width: 100%; max-width: 400px; padding: 20px; margin: 0 auto;">  
    <img src="/img/2026-02-06-features-released-january-february/error.gif"

For situations where there's no error alert, users can access this feature by clicking "More" on the home page, then selecting "Upload app info." This ensures comprehensive support coverage for all types of issues. Refer [here](https://avni.readme.io/docs/troubleshooting-and-error-debugging#reporting-an-issue) for more details.

### ⚡ Enhancements in Data Upload

- Over 50% reduction in upload time for bundles with more concepts (5k+)
- Made the process for registration via CSV uploads more intuitive for organisations with multiple location hierarchies
